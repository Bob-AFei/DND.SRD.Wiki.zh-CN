# DND.SRD.Wiki.zh-CN

[![License: MIT](https://img.shields.io/badge/License-MIT-green.svg)](./LICENSE)

DND.SRD.Wiki.zh-CN 是一个面向中文社区的 **D&D 5e System Reference Document（SRD）** 中文翻译项目。项目以英文版 [DND.SRD.Wiki](https://github.com/OldManUmby/DND.SRD.Wiki) 为基础，目标是将其 Markdown 化的 SRD 内容系统性翻译为中文，在保证术语统一、结构清晰与可持续维护的前提下，构建一套开源、免费、便于检索与再利用的中文规则参考资料库。

本仓库面向玩家、DM、内容创作者、规则整理者与开发者，强调：**术语准确、结构完整、格式稳定、便于部署、开放协作**。

---

## 1. 项目概述

本项目围绕 D&D 5e SRD 的公开内容展开，计划对英文原版 Wiki 中约 **1,059 个 Markdown 文件**进行系统性中文化整理。翻译工作以 **SRD_CC_v5.2.1** 为最终权威来源，以英文 DND.SRD.Wiki 为格式与内容结构参考，以 DND5e_chm 为中文术语参考，尽可能保证中文表达自然、规则含义准确、章节结构与原始内容保持一致。

## 2. 项目愿景（Vision）

本项目希望为中文 D&D 社区提供一份高质量、术语统一、可自由获取和再利用的 SRD 中文参考资料。

我们的长期愿景包括：

1. 为中文玩家、DM 与创作者提供稳定可靠的规则入口。
2. 降低中文用户阅读英文 SRD 的门槛，帮助更多人顺利进入 D&D 5e 体系。
3. 在翻译实践中沉淀一套标准化、可复用的 D&D 5e 中文术语体系。
4. 通过开源协作形成可持续维护的社区项目，而不是一次性整理成果。
5. 为后续 Wiki、站点、检索工具、规则数据库等应用提供结构化 Markdown 内容基础。

---

## 3. 项目目的（Purpose）

本项目的直接目标如下：

- 系统性翻译 DND.SRD.Wiki 的全部英文内容。
- 严格参考 DND5e_chm 等官方中文资料，尽量保证术语准确与风格统一。
- 保留原始 Markdown 结构，以便后续用于 Wiki、站点发布、静态生成与知识库整理。
- 建立清晰的目录与命名规范，方便多人协作与长期维护。
- 以开源方式推进项目，接受社区贡献、校对与术语修订建议。

### 项目原则

- 忠于规则原意，不擅自改写规则效果。
- 优先统一术语，其次优化语句流畅度。
- 保持结构稳定，避免破坏原始 Markdown 层级。
- 翻译结果应既适合阅读，也适合程序化处理。

## 6. 翻译规范（Translation Standards）

为保证项目整体质量，所有内容建议遵循以下翻译规范。

### 6.1 术语一致性原则

- 同一术语在相同语境下应保持统一译法。
- 专有名词优先参考官方中文资料与术语对照表。
- 不随意创造新译名，不使用缺乏依据的音译或意译。
- 如遇多版本译名冲突，应在项目内统一后再批量使用。

### 6.2 翻译来源优先级

翻译时遵循以下资源优先级，以保证内容的权威性与准确性：

1. **SRD_CC_v5.2.1.md** — 最权威来源，所有翻译文本最终以此为准
2. **DND.SRD.Wiki/** — Markdown 格式与内容结构参考
3. **DND5e_chm/** — 中文术语参考

翻译完成后，必须对照 **SRD_CC_v5.2.1.md** 进行最终校对，确保内容与 SRD 官方文档一致。

### 6.3 标题格式

原则上采用“中文名 + 英文名”的双语标题格式，便于中文阅读与英文检索。

示例：

- `# 战士 Fighter`
- `## 法术位 Spell Slots`
- `## 火球术 Fireball`

### 6.4 Markdown 格式保持

- 尽量保留原始标题层级。
- 保持列表、表格、引用块、分隔线等结构不变。
- 不随意调整段落顺序。
- 仅在必要时为中文可读性做最小限度的格式修正。

### 6.5 度量单位处理

项目当前采用便于中文读者理解的单位表达方式：

- `feet` → `尺`
- `pounds` → `磅`

### 6.6 通用缩写保留

以下常见规则缩写建议保留英文大写形式，不强行翻译：

- AC
- HP
- DC
- XP
- CR
- STR / DEX / CON / INT / WIS / CHA

### 6.7 语言风格建议

- 语气专业、清晰、简洁。
- 规则句尽量直译其逻辑关系，避免文学化改写。
- 描述性文本可适度润色，但不得偏离原意。
- 同一类别内容尽量保持一致的句式与节奏。

### 6.8 翻译后对比检查

每个翻译任务完成后，需自动对比 DND5e_chm 目录中的中文参考资料，进行完整性检查和术语校准。

**检查流程：**

1. 根据翻译类别，定位 DND5e_chm 中对应的中文参考文件。
2. 逐项对比翻译内容与中文参考，重点关注专业术语一致性、属性描述完整性和关键信息完整性。
3. 如发现更准确的术语表达，采用 DND5e_chm 的译法；如发现信息缺失，参照 SRD 原文补充。
4. 检查完成后，简要记录与参考内容的一致性情况和已采纳的修正。

**对比优先级原则：**

- 内容准确性：以 SRD_CC_v5.2.1.md 为最终权威
- 术语表达：优先采用 DND5e_chm 的标准译法
- 信息完整性：两者结合，确保 SRD 范围内的所有信息均已翻译
- 格式规范：以 DND.SRD.Wiki 的 Markdown 格式为准

> **注意：** DND5e_chm 可能包含非 SRD 范围的扩展内容，对比补充时应仅限于 SRD 核心内容。

## 9. 参与贡献（Contributing）

欢迎任何形式的贡献，包括但不限于：

- 新文件翻译
- 现有译文校对
- 术语统一修正
- Markdown 结构修复
- 链接、命名、目录映射优化

### 参与流程

1. Fork 本仓库。
2. 创建分支并进行翻译或修订。
3. 自查术语与 Markdown 格式。
4. 提交 Pull Request。
5. 等待审阅与合并。

### 贡献提醒

- 提交前请尽量确认术语与既有译法一致。
- 优先保持原始结构，不要一次性混入大规模格式重排。
- 若涉及批量术语修改，建议在说明中标注修改依据。
- 对存在争议的译名，可先通过 Issue 或讨论区提出建议。
